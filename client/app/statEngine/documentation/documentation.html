Overview

What is StatEngine?

StatEngine is a real-time open source data analytics and visualization platform built from the ground up for public safety organizations. StatEngine is built on powerful, best-of-breed, open source analytics and visualization tools. This means public safety organizations of all sizes can easily and cost-effectively exploit large volumes of complex, highly relevant, and impactful data sets. StatEngine is currently funded by the Public Safety Innovation Accelerator Program (PSIAP).

Using This Guide

This manual is intended for system administrators responsible for installing and managing StatEngine data. We’ve included a quickstart that will cover the majority of implementations, but also provided more detailed documentation that explains all the features and will help for more customized implementations.

Getting Started

1. Connect Your Data

StatEngine integrates with common fire department data sources including data warehouses, Computer Aided Dispatch (CAD) systems and Records Management Systems (RMS). Multiple approaches, listed below, can be used to securely send data to StatEngine. The department should review the various approaches and once the desired integration approach is identified, fill our our questionaire here.

Spade

Spade can be run on your local network to read data from your local database(s) and send it to StatEngine. Spade can also be configured to run on your local network and watch a directory for CAD files and then push them to NFORS.

PulsePoint

If your department has an existing PulsePoint integration, you can leverage the same integration to send CAD data to StatEngine.

Linked Database

If you are able to push data to a database in our cloud environment, we can provide you with connection information to push data to a dedicated database we stand up for you. You can find more information on linking MSSQL databases here. This approach minimizes the amount of software that needs to run on a department’s infrastructure, however it requires the local jurisdiction to setup and configure the linked database and synchronization logic.

2. Verify Your Data

Once StatEngine starts receiving data, we will work with you to identify missing or erroneous data. Often times we'll need details on missing apparatus types, battalion and district mappings to enrich your data to the fullest extent.

3. Access Your Data

After data verfication, we will set you up with a customized dashboard so you can start exploring your data. Check out what's possible here.

4. Explore Your Data Even More

StatEngine has even more ways to add and enrich your data. Explore plugins like adding weather, traffic, or directly tweeting stats to Twitter. Start exploring the Marketplace

Dashboard Features

These are just a few visualizations that come built-in to your dashboard, but you have the ability to create new and custom ones. The possibilities are endless!

Basic Stats

Get quick incident stats, response times, and more for yesterday, last week, last year, or even a custom time period.

Complex Graphs

Plot which units respond to the most calls, how each unit is utilized across call types, or both! Bar graphs, pie charts, and more!

Advanced Time Series

Perform advanced time series analysis such as averaging unit response time or visualizing which hour of the day is most active.

Geo Data

StatEngine plots all incidents on a map. See where your most active areas are!

Frequently Asked Questions

Who created statEngine?

StatEngine was created and is maintained by Prominent Edge. Check out prominentedge.com to find out more.

Is my data secure?

Yes, all data transferred is through an encrypted HTTPS session. This is the same security that is used by your bank when you access your account online. Once, your data is on the StatEngine servers it is stored using industry best practices.

Who manages the StatEngine installation?

Departments are responsible for installing and managing StatEngine. However, if you have any issues you can contact support@statengine.io and we’ll be happy to help you.

Is there any prerequisite software that would need to be installed for StatEngine to work?

No, StatEngine doesn't have any dependencies. For optimal experience, we suggest using Google Chrome, but all modern browsers are supported.

Who do I contact for support if there is an issue with the application?

You can contact support@statengine.io.
