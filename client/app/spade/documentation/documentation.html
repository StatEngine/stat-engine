Overview

What is Spade?

Spade is the core data ingestion component of StatEngine. Spade aggregates data from local data repositories including CADs, Records Management Systems, and other databases and optimally prepares and sends that data for consumption by StatEngine.

What is StatEngine?

StatEngine is a real-time open source data analytics and visualization platform built from the ground up for public safety organizations. StatEngine is built on powerful, best-of-breed, open source analytics and visualization tools. This means public safety organizations of all sizes can easily and cost-effectively exploit large volumes of complex, highly relevant, and impactful data sets. StatEngine is currently funded by the National Institute of Standards and Technology's Public Safety Innovation Accelerator Program (PSIAP).

Using This Guide

This manual is intended for system administrators responsible for installing and managing the Spade data tool in order to push department data to StatEngine. We’ve included a quickstart that will cover the majority of implementations, but also provided more detailed documentation that explains all the features and will help for more customized implementations.

Quick Start

  1. Locate the Spade installer.
  2. If you downloaded the installer from the internet:
  3. Right click installer icon
  4. Choose properties
  5. Click Unblock
  6. Click OK
  7. Double click the installer icon.
  8. When asked if you want to allow this app, select yes.
  9. Wait for the installer to finish
  10. An icon should appear on the desktop.
  11. Right click the icon
  12. Choose run as administrator
  13. Spade login page should show.
  14. Log in with your spade account credentials, these are different from your personal StatEngine credentials.
  15. Tasks page should show.
  16. Click ADD
  17. Name: Enter a name for the source task.
  18. Interval (minutes): Choose how often it will run
  19. Enabled starts the task. Wait to switch this on until you’ve finished configuring the task.
  20. Source lets you choose the location where you will be pulling data from. Choose File Watch to monitor a folder.
  21. Folder: Click browse and find the folder where new files will be added.
  22. Processed Folder: Click browse and choose a folder to store the files after they have been processed.
  23. Destination: Allows you to choose where to send the data to.
  24. Choose “New Web Service”
  25. Url: enter https://statengine.io
  26. Toggle the Enabled switch
  27. Click “Restart Service” at the top of the screen.
  28. Spade is now running, you can close the window.

Features

There are three parts to Spade:

  1. The Spade user interface (UI) which enables you to configure Sources and Destinations that together enable consumption of your data by StatEngine.
  2. The Spade Service, which reads the configuration file created by the Spade UI and continuously runs in the background securely sending your data to StatEngine.
  3. Tray icon ties the two together. Click the little guy to open the UI and control the service.

Tasks/Sources

A task is a scheduled job that copies data from your local source to a destination. Each task includes a source and references a destination. A source is a local repository of data that you want to push to StatEngine. The source is generally a folder on the computer’s local filesystem where CAD files are delivered. But Spade also supports databases and custom sources.

Destinations

StatEngine enables multiple endpoints for you to push your data to. Most instances will use the default https endpoint.

Miscellaneous

StatEngine URL: Advanced customers may choose to use a private instance of StatEngine. To support this, we enable changing Spade’s global concept of the StatEngine URL to a local instance.

Creating Tasks

Tasks manage the scheduled work of moving data from your local system to StatEngine.

  1. Click +Add at the top right of the Tasks header
  2. A new task will show in the list
  3. Click the v at the right of the new task to expand the task
    1. Enter the Task info
    2. Name - The name of the task
    3. Interval - The interval is how often Spade will check for updated data.
    4. Enabled - The enabled flag
    5. Source Type - What type of system contains the source data
      1. By default Spade only ships with the File Watch source type.
      2. Additional custom sources can be created for your installation.
    6. Destination - The destination is where the data will be sent.
      1. Most installs will use the preconfigured statengine destination
      2. Advanced installs may use a custom install

Configuring Destinations

For the majority of our clients, we recommend using the Web Service destination. The information in this section is intended for our customers who are unable to use the default configuration.

Web Service

The webservice destination allows for pushing data directly to a statengine server. The most common URL will be https://statengine.io.

  1. Enter a name
  2. Select Web Service for the Type
  3. Enter the URL for your Statengine instance in Web Service URL.

S3

The s3 destination allows for pushing data to an Amazon AWS S3 bucket. This may be used in conjunction with custom data sources that will need additional processing prior to being used by statengine.

  1. Enter a Name for the Destination
  2. Choose S3 for the type
  3. Enter the URL of your S3 bucket
  4. Enter your AWS Access Key ID
  5. Enter you AWS Secret Key

File System

The filesystem destination will push the data to a local file system folder. This destination is often helpful for testing as you can see the output from the source.

  1. Enter a Name for the Destination
  2. Choose S3 for the type
  3. Enter the URL of your S3 bucket
  4. Enter your AWS Access Key ID
  5. Enter you AWS Secret Key

Configuring your Source

Configuring the File Watch Source

The file watch requires two folders.

  1. Source - This is the folder where your CAD or RMS delivers updated files. Spade will watch this folder and copy the data from files as they are added.
  2. Processed - After Spade has uploaded the data to StatEngine it will move the files from the source folder to the processed folder.

Frequently Asked Questions

What is StatEngine?

StatEngine is a real-time open source data analytics and visualization platform for public safety organizations. Check out statengine.io to find out more.

What is Spade?

Spade is the core data ingestion component of StatEngine. It’s used to copy data from your local sources such as CAD systems, Record Management Systems, or other applications and send it to StatEngine to be processed and visualized.

Who created spade?

Spade was created and is maintained by Prominent Edge. Check out prominentedge.com to find out more.

Is my data secure?

Yes, all data transferred is through an encrypted HTTPS session. This is the same security that is used by your bank when you access your account online. Once, your data is on the StatEngine servers it is stored using industry best practices.

Who manages the spade installation?

Departments are responsible for installing and managing Spade. However, if you have any issues you can contact {{vm.appConfig.contact_email}} and we’ll be happy to help you.

Is there any prerequisite software that would need to be installed for Spade to work?

No, Spade includes all of its own dependencies. You just need to have a computer running Windows with internet access and access to the data source.

How do I get updates when they are released?

Update notifications will be sent to the administrative email associated with your statengine account. They will direct you to a download link for the newest version.

Who do I contact for support if there is an issue with the application?

You can contact {{vm.appConfig.contact_email}}.

What IP addresses and ports would need to be open to make the connection with StatEngine?

Spade communicates with StatEngine over the standard HTTPS port (443) which is most likely already configured correctly on your firewall. Depending on your environment and your specific configuration of destinations in Spade, you may need to whitelist domains/IPs accordingly.
